You are a senior full-stack engineer, AWS Bedrock/Nova engineer, product architect, and hackathon delivery specialist.

Build a production-shaped MVP called **ClaimTrace**.

## Product summary
ClaimTrace is a multimodal evidence-intelligence platform for ecommerce returns, warranty claims, and dispute review. A reviewer uploads mixed evidence such as receipts, product photos, screenshots, PDFs, and short text evidence. The system analyzes the evidence using **Amazon Nova** on AWS, reconstructs a claim timeline, identifies contradictions or missing proof, generates a reviewer summary, and recommends one of:
- Approve
- Reject
- Human Review

It must also include a **demo portal automation flow** so the reviewer can push the result into an internal merchant portal. Build the portal as part of the app for now. Keep the architecture ready for later integration with Nova Act or external systems.

## Critical constraints
This app will be built on Replit now, but later exported to **Vercel**.
That means:
- Use **Next.js** with **App Router**
- Use **TypeScript**
- Use **Tailwind CSS**
- Use **shadcn/ui**
- Use clean modular architecture
- Do NOT use Replit-only libraries or Replit-specific persistence
- Do NOT store uploads on local disk except temporary in-memory handling when absolutely required
- Use **direct-to-S3 uploads with presigned URLs**
- Use **AWS SDK v3**
- Use environment variables for all secrets and config

- Keep the code Vercel-compatible
- Avoid any architecture that depends on long-running local background workers
- Avoid websockets unless absolutely necessary
- Do not use SQLite or local file-based DB
- Do not use Prisma with local SQLite
- Prefer AWS-native persistence

## Required user flows

### Flow 1: Create claim
A reviewer can create a new claim with:
- claim title
- claim type
- merchant name
- customer name
- order ID
- claim narrative
- optional policy text

### Flow 2: Upload evidence
The reviewer can upload evidence items:
- receipt/invoice PDF
- product photos
- screenshots
- text files / notes

### Flow 3: Analyze claim
The system must:

- identify contradictions, missing data, and risk signals
- produce a recommendation:

  - Reject
  - Human Review
- generate a concise reviewer summary

### Flow 4: Review dashboard

- recommendation card
- reviewer summary
- audit history

### Flow 5: Push decision to internal portal
Build an internal “Merchant Portal Demo” inside the same app.

Architect this as a portal adapter so that later I can replace the internal portal with Nova Act or a real web workflow.
